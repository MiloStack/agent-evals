What It Does

D

DevOps domain

Catch missing tests, leaked secrets, TODO/FIXME leftovers, oversized files, and weak error handling before merges land.

S

Security domain

Flag insecure HTTP, disabled TLS verification, dangerous shell patterns, weak crypto, and wildcard CORS policies.

P

Performance domain

Detect innerHTML abuse, blocking sync loops, N+1 query patterns, memory leaks, and DOM thrashing.

CI

GitHub Action ready

Run agent-evals in pull requests with path, domain, and strict-mode inputs so CI fails when high-risk output slips through.

R

Versioned release

Public GitHub release v0.1.0 is live now, giving early adopters a stable milestone to test and reference.

Installation & Usage

Global Install
npm install -g agent-evals
Run Evaluation
npx agent-evals evaluate ./src --domain security --strict
Run in GitHub Actions
uses: MiloStack/agent-evals@v0.1.0
